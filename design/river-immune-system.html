River's Immune System

A defense engine that keeps River actually working, not just running. River's current self-healer (PR #99) is an excellent crash detector — but in 24 catalogued production incidents, crashes accounted for zero of them. The pain is entirely in failures where River was up, healthy, and wrong: 8 days silently on the metered API, deep_search hollow for 11 commits, a 33-minute invisible Signal socket during a live meetup, a wrong calendar URL for weeks. This doc designs the missing sense organ: a ProbeRegistry of deterministic antibodies (each past scar becomes a permanent probe), a boot-time config-assertion layer that fails loud instead of quiet, and — gated hard behind the existing #110 safety rails — an agentic remediation layer where River diagnoses and repairs from an allowlisted action set. The immune memory compounds: every new break adds one row and River never takes that wound silently again.

Honest status: nothing here is built. This is a design grounded in two evidence artifacts (a 24-incident failure catalogue and a full current-coverage map, both mined this session from the repo + memory graph). The incident catalogue is archaeology from memory files and git log — durations and root causes are best-effort, not audited. The agentic-remediation layer is a real trust boundary + blast-radius surface (an agent with world-acting hands, processing partly attacker-influenceable relay traffic) and its safety gates are a merge prerequisite, not a follow-up — see Risks. Build order is forced: you cannot remediate what you cannot sense, so the sensing spine ships and is backtested against the 24 incidents before River is given hands.

Purpose

The Night-Loop Spine opens by declaring itself the offense engine and dismissing "watch, detect, audit, reconcile" as mere defense. This is that defense — and it turns out defense is not a solved problem River can take for granted. River breaks in production often, and the breaks that hurt are precisely the ones his current healer cannot see. The two engines are the poles of one nervous system: the night-loop manufactures what should exist; the immune system keeps what exists alive. They can eventually share a substrate — an antibody is a "standing fire" that glows when its probe trips.

Who/what it is for: River (the dreamfinder bot) in production on OCI, plus its bridge mesh. The beneficiary of a working immune system is Nick — it removes his current job of being River's failure detector (he found the metered drift, the hollow deep_search, and the empty calendar by hand). This is design-for-subtraction: the human stops being the monitor.

Architecture: three layers + an immune memory

The design mirrors a nervous system. Autonomic reflexes are deterministic and always-on; conscious self-examination is periodic and agentic. Detection is never agentic (an LLM sensor can be fooled by River's own hallucination); only diagnosis and repair are.

Layer 0 — Autonomic reflexes (deterministic, boot + always-on)

Extends what exists. The Docker-healthcheck→restart arc stays. New: boot-time config assertions that fail loud. River refuses to come up quiet if a startup invariant is violated:

assert authMode == OAuth            // kills the 8-day metered drift at t=0
assert calendar.events.length > 0  // kills the weeks-long wrong-URL calendar
assert env(critical).allPresent    // semantic, not just non-null
assert deepSearchProbe.nonHollow   // integration-hollow caught at boot
Fail loud, not fail quiet. Every incident in class (b)/(c) survived because a failure was swallowed into a clean-looking state. A violated boot assertion escalates to Telegram and is surfaced in /health before River accepts traffic. A config that can't be verified withholds readiness — uncertainty removes authority, it does not grant it.

Layer 1 — Behavioral senses (deterministic probing — the missing organ)

A ProbeRegistry: a declarative library where each entry is:

Probe {
  id:            "deep_search_lore",
  invocation:    () => deepSearch("<known lore question>"),
  invariant:     (r) => r.contains(KNOWN_SENTINEL),   // deterministic assert, NO LLM
  failureClass:  "integration-hollow",
  remediation:   Action.reVendorClis,                 // Layer 2 hook (see safety)
  autonomy:      AutonomyLevel.detectOnly,            // per-probe, escalates by default
}

On a self-check tick (riding the #16 heartbeat, not a new cron), River actually invokes his own real paths with canned inputs that have known-non-empty answers, and asserts the result isn't hollow. Per-probe status is published in /health — so silent-semantic failure becomes visible for the first time.

  • Honest health = failure rate, not last-success. Incident #8 survived because /health asked "did the last call eventually succeed?" A probe asks "what fraction is hollow right now?"
  • Config truth = the deployed artifact, not the repo or memory. Incidents #9 and #19 both passed code review — the code was correct, the running container was wrong. Config antibodies read the live environment (in-process env, or a constrained readback), never the merged .env.
  • Probes are cheap and bounded. Each has a hard timeout and runs off the message path; a slow probe degrades to "unknown", never blocks user traffic.

Layer 2 — Agentic diagnosis + remediation (River as his own doctor)

When a probe trips, River wakes a bounded self-heal session with the failing probe + context, diagnoses, and picks a remediation from an allowlisted action set — not arbitrary shell. River chooses which named action to run; he never authors the action. This is the cage: blast radius is the allowlist, not the shell.

enum Action {                 // the ENTIRE surface River can touch
  reVendorClis,               // re-run scripts/vendor-clis.sh, re-verify manifest
  repointConfig(key, value),  // from a bounded, pre-approved key→value allowlist
  restartConnection(name),    // a named bridge/MCP connection reconnect
  clearCorruptHistory,        // already exists in the sync loop
  restartContainer(name),     // ⚠ highest blast radius — see Risks; gated hardest
  escalateOnly,               // the safe default for anything unrecognised
}

Every remediation passes the #110 safety rails already built for Community Spark:

  • Single-action CAS latch — at most one heal in flight (partial unique invariant, same pattern as single-pending-draft).
  • Notify-backstop — every autonomous action POSTs to the notify sidecar before/as it acts, so Nick sees each heal in real time.
  • Circuit-breaker — after N failed self-heals of the same probe, River stops thrashing, escalates, and goes hands-off on that probe. A guard that can't confirm success must withhold the capability, not keep firing it.
  • Fail-closed on the unknown — an unrecognised failure or an out-of-allowlist fix resolves to escalateOnly, never to an improvised action.

The immune memory (why this compounds)

Because the antibody library is declarative, every future break becomes a one-row addition: new scar → new Probe → River never takes that wound silently again. It is a regression suite for production reality that lives inside River and grows monotonically. For a golem who inspects his own inscription, immune memory is not a metaphor — it is the data structure. This is the property that makes the healer brilliant rather than merely thorough: it learns the shape of each wound and files an antibody.

The backtest (the falsifier, and the cage-before-monster gate)

Before River is given any hands, the sensing spine is validated against ground truth: replay each of the 24 catalogued incidents and confirm its probe would have tripped. A probe that does not catch its own historical incident is a probe that does not work — caught on paper, not in production. This is the concrete form of "cage before monster": the senses must be proven discriminating (they fire on real failures) and specific (they don't false-positive on a healthy system) before the remediation layer is wired to anything.

Instrument-not-blind discipline: for each probe we also name the failure it cannot see. A deep_search sentinel probe is blind to a subtly-wrong-but-non-empty answer; a calendar > 0 probe is blind to a calendar showing the wrong events. Those residual blind spots are logged as future antibodies, not hidden.

Invariants & Guarantees (as built)

  • Detection is deterministic; only diagnosis/repair is agentic. No LLM sits in the sensing path — a probe is an assertion, so River's own hallucination can never mark a broken tool healthy.
  • River's hands are an allowlist, not a shell. Remediation is a fixed enum of named actions; anything outside it resolves to escalateOnly. Blast radius is the allowlist size.
  • Every autonomous action is single-latched, notify-backstopped, and circuit-broken. The #110 rails are reused verbatim as the policy layer; the healer adds no new unguarded write path.
  • Sensing ships and is backtested before hands are wired. The remediation layer has nothing to act on until Layer 1 exists and has passed the incident backtest. This is the invariant that keeps the doc's own central risk (an agent with hands on partly-untrusted input) from being the first thing shipped.
  • Fail loud is the default disposition. Anything the immune system cannot verify becomes a visible degraded/escalation, never a swallowed clean state.

Designed, not built

Status: the entire document is a design. No probe, registry, boot assertion, remediation action, or backtest harness exists yet. The evidence base is two scout reports produced this session; the incident catalogue is archaeology, not an audited log.
  • The ProbeRegistry & every antibody. Concept only. The exact sentinel questions, invariants, and timeouts are unwritten.
  • Boot-time config assertions. Designed; the readback mechanism for "live deployed artifact" (in-process env vs a constrained container readback) is unspecified and interacts with the container's trust boundary.
  • The allowlisted action set. Named, not implemented. restartContainer in particular has an unsolved capability question (see Risks) — River does not currently have, and should not casually be given, Docker-socket access.
  • Riding the #16 heartbeat. The heartbeat itself is a separate open design (task #16); until it exists the self-check tick would need a temporary home in the existing 60s scheduler.
  • The backtest harness. The replay-24-incidents falsifier is the plan; the fixtures don't exist.

Risks & Limitations

  • High (BLOCKING for autonomy) — An agent with world-acting hands on partly attacker-influenceable input. River processes messages from a multi-platform relay mesh (Signal/Telegram/WhatsApp/Discord via puppets); that input is attacker-influenceable, and the remediation layer lets River act on the host. Prompt-injection into a self-heal decision is the threat. Mitigation is a merge prerequisite, not a follow-up: the diagnosis prompt must be fed only deterministic probe results (never raw user text), the action set is a closed allowlist, and every action is notify-backstopped so a wrong heal is visible immediately. Cage-match by law.
  • High (BLOCKING for restartContainer) — Container-restart capability is a large blast radius with no safe primitive today. Restarting mautrix-signal (incident #3's fix) requires Docker-socket access; mounting the Docker socket into River's container is a well-known container-escape-equivalent risk. This action must NOT ship as in-process Docker access. Options to specify before enabling it: a constrained side-car that accepts only a fixed set of restart <named-container> requests over an authenticated local channel, or leaving container-restart as escalateOnly permanently. Until resolved, #6's auto-heal stays detect-and-escalate.
  • High — A self-healer can mask the very failures it should surface. If River silently re-vendors a CLI or repoints a URL every hour, a persistent upstream breakage looks healthy while thrashing underneath. The circuit-breaker (auto-pause + escalate after N heals) is the guard, and every heal is logged/notified — a heal is never silent. A rising heal-rate is itself a monitored signal.
  • Med — Probes add production load and a new failure surface. Each probe invokes a real path; a buggy or slow probe could degrade the bot. Mitigation: hard per-probe timeouts, off-message-path execution, and a global kill-switch for the whole self-check tick.
  • Med — Sentinel probes are blind to subtly-wrong output. deep_search returning the sentinel proves non-hollow, not correct; a calendar with >0 wrong events passes. These residual blind spots (see backtest) are logged as future antibodies, not claimed as covered.
  • Med — Config-readback of the "live artifact" may itself need container introspection. Reading the truly-deployed env (vs the in-process view) can require the same privileged readback the restart action does. Scope the boot-assert to what River can see in-process first; escalate the rest.
  • Med — The incident catalogue is archaeology. Durations, root causes, and counts were mined from memory + git this session, not from an audited incident log. The histogram's shape (silent-semantic dominant, crashes zero) is robust; individual rows may be imprecise.
  • Low — Antibody rot. A sentinel question ("known lore answer") can go stale if the underlying data changes, turning a probe into a false alarm. Probes that depend on mutable data need their sentinels versioned alongside the data.
