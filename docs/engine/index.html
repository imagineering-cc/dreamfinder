The Dreamfinder Engine

A getting-started guide for building your own bot or avatar on the same machinery that runs River. Written for Andy.

Dreamfinder (the character is named River) is two programs that have never shared a line of code: a brain that thinks in chat, and a body that speaks and moves in 3D. Each one has a small, genuinely reusable core wrapped in a lot of River-specific decoration. This guide's whole job is to show you where the core ends and the decoration begins, so you can keep the core and replace the rest with your character.

The one idea to take away. An "engine" is something you depend on and configure, not something you fork and edit. Dreamfinder today is about 80% of the way there: the seams exist and are clean, but they are held together by convention rather than packaged as libraries. This guide shows you how to use the seams that exist now, and the roadmap page is an honest account of the handful of extractions that would make it a true drop-in engine.

The two halves

🧠

The Brain

Dart. A chat-agent engine: message in → Claude call with tools → reply out. Platform-agnostic, LLM-agnostic, tool-agnostic. Runs on Matrix today; the agent loop does not know Matrix exists.

Read this to: build a chat/PM bot on any platform.
✨

The Body

JavaScript + Three.js. A voice + 3D avatar engine: audio in → speech → lip-synced talking head out. Two voice backends behind one interface, hot-swappable mid-sentence.

Read this to: embed a talking avatar anywhere.

They meet at exactly one place: the body sends a line of text to the brain over HTTP and gets a line of text back to speak. That accidental boundary is the cleanest interface in the whole system. We call it the Brain API, and it is what lets you mix and match - your brain with River's body, River's brain with your avatar, or both replaced.

What counts as "the engine"

Throughout the guide, two tags mark every component:

engine reusable core - keep it, configure it.
glue River-specific - this is where your character's code goes instead.

Halfengineglue
Brain lib/src/agent/ (AgentLoop, ToolRegistry, ConversationHistory, system-prompt builder), lib/src/mcp/, lib/src/db/ everything in lib/src/tools/, the 1090-line bin/dreamfinder.dart wiring, the Matrix client

Pick your path

I want to build my own chat bot (different platform, different character)

Go to The Brain. You will implement two small interfaces (an LLM call and a platform client), register your own tools, and write a system prompt. The agent loop, conversation history, tool routing, and MCP integration come for free. The minimal-bot sketch is ~40 lines.

I want a talking 3D avatar on my own site or event

Go to The Body. The fastest path is the standalone avatar-renderer.js - drop it in an iframe, feed it PCM audio and a mood string, and it lip-syncs. The full path is swapping the voice substrate to your own STT/TTS/LLM behind the startVoicePipeline interface.

I want both - my own embodied character, end to end

Read both leaf pages, then the Brain API section: that is the contract your brain must speak so any body can drive it. Build the brain first (you can test it in chat with no avatar), then point a body at it.

I just want to understand how River works to contribute

Read the two leaf pages top to bottom. The roadmap doubles as a map of the codebase's current rough edges - those are the best places to make a first contribution.
